Project Overview

  • Goal: Integrated test system for automotive safety components
  • Web-based interface for operation and analysis
  • Custom-built hardware, firmware, and software solution
  • Multiple communication protocols: PWM, Analog, UART, OpenAMP & IPCC
  • Streamlined field claim evaluation process
- Explain what the project was - We built a system that tests automotive safety components primarily in a lab setting for ZF Group's Claims Investigation Committee with the goal of streamlining the field claim evaluation process. - The system is designed to be modular and flexible to accommodate a variety of safety components including brake signal transmitters, wear sensors, pressure sensors, and electronic stability control modules, all part of ZF Group's product line. - The system is designed to be user-friendly and accessible to a wide range of users, including automotive safety component manufacturers, suppliers, and testing facilities.

Devices Under Test

Brake Signal Transmitter (BST)

  • Two PWM outputs (0.7V - 4.5V)
  • Linear stroke measurement (0-8mm)
  • Operating voltage: 8-32V DC
  • Switch signal for system wake-up

Functional Overview

Acts as the "brain" that reads how hard a driver presses the brake pedal. Essential for:

  • Converting pedal movement into electronic signals
  • Telling the brake system how much braking force is needed
  • Ensuring smooth and responsive braking

Optional string potentiometer for precise stroke validation

Pressure Sensor

  • Analog output: 0.5V - 4.5V DC
  • Pressure range: 0-10 bar
  • Operating voltage: 8-32V DC

Functional Overview

Sensor that continuously measures relative pressure in vehicle control systems. Essential for:

  • Brake system pressure monitoring
  • Pneumatic system diagnostics
  • Safety system validation

Continuous Wear Sensor

  • Analog output: 0.7V - 4.0V DC
  • Operating voltage: 5V DC
  • Measurement range: 37mm

Functional Overview

Works like a monitor for your brake pads and discs. Helps with:

  • Warning when brakes are wearing down
  • Predicting when maintenance will be needed
  • Preventing unexpected brake failures

Test Signal Requirements

  • Analog Voltage Measurement (Potentiometer, Pressure & Wear)

Building a Custom Embedded Linux Distribution with the Yocto Project

  • The Yocto Project is a collection of open-source tools for making a custom embedded linux distibution
    • A distribution in Linux is an operating system that uses the Linux Kernel
    • In this case, we only take the Linux kernel and build the necessary software needed for our application
  • OpenEmbedded - build engine that provide tools for cross-compilation for various architectures
    • Provides essential metadata (software, source code, machine specific configuration)
    • Also provides bitbake, build tool responsible for parsing metadata as recipes and initializing build environment so it can build from source all the software to be deployed on the image

Using Yocto to Build a Linux Distribution for the STM32MP157F-DK2

  • Software to connect to Wi-Fi (wpa_supplicant, and implementation of an IEEE 802.11i supplicant)
  • Webserver to host a website (nginx)

Inter-Processor Communication (IPC)

With a heterogenous architecture, (Cortex-M4 and Cortex A7 share same memory) how is information shared?
With OpenAMP!
  • OpenAMP (Open Asymmetric Multi-Processing) is a response to the increasing amount of heterogenous architectures
    • Heterogenous architectures are systems that contains multiple processors with different instruction set
  • Provides software framework that places a standard protocol for the shared memory to allow for easier communication
  • Key component for Cortex-M4 (microcontroller) is libmetal, an API that handles device interrupts and memory accessing in order to properly communicate to Linux
  • In the Linux kernel (Cortex-A7), Remote Processor Messaging (rpmsg) is the used shared-memory based communication protocol
  • On both processors, these communication protocols are not directly to each other, but implement the VirtIO open standard, which buffer the messages between the two in a shared memory space
